# AI-README — เริ่มอ่านที่นี่ (สำหรับ AI ทุกตัวที่มาทำงานใน repo นี้)

โปรเจกต์นี้คือ **Ener Scan** — LINE OA บริการสแกนพลังพระเครื่อง/เครื่องราง/กำไลหิน (สายมู)
เจ้าของ: กบ (คุยไทย ชอบตรง ๆ สั้น ๆ) · โดเมน: scan.my-ener.uk (prod) / test.my-ener.uk (staging)

## ลำดับการอ่านก่อนเริ่มงาน (บังคับ)
1. `docs/ai/STATE.md` — ระบบคืออะไร อะไร LIVE อยู่ วิธี deploy กติกาห้ามแตะ
2. `docs/ai/LOG.md` — อ่านย้อน 3-5 รายการล่าสุด จะรู้ว่างานเดินถึงไหน ค้างอะไร
3. `docs/ai/BACKLOG.md` — คิวงานทั้งหมดพร้อมสถานะ

## กติกาบังคับตอนจบงาน (ทุก AI ทุกครั้ง)
- งานเสร็จ (หรือหยุดกลางคัน) → **ต้องเติมบรรทัดใน `docs/ai/LOG.md`** ระบุ วันที่ | ทำอะไร | ผล | ค้างอะไร
- ถ้างานทำให้สถานะระบบเปลี่ยน (feature ใหม่ LIVE, env ใหม่, กติกาใหม่) → อัปเดต `docs/ai/STATE.md` ด้วย
- ถ้างานในคิวเสร็จ/เกิดงานใหม่ → ย้าย/เพิ่มใน `docs/ai/BACKLOG.md`
- commit ไฟล์บันทึกพวกนี้ไปพร้อมงาน — ห้ามจบงานโดยไม่จด

## กติกาความปลอดภัย (ห้ามฝ่าฝืน)
- ห้ามอ่าน/พิมพ์ค่าจริงใน `.env` ของ prod — ดูได้แค่ชื่อตัวแปร
- Deploy **pro ต้องให้กบสั่งชัด ๆ เท่านั้น** / staging สั่งได้เมื่องานพร้อม
- ห้าม `git checkout -- .` หรือ commit ไฟล์ compose บนเครื่อง server (compose บน server ถูกแก้เฉพาะเครื่อง)
- การลบ/แก้ข้อมูลลูกค้าใน DB ต้องให้กบยืนยันก่อนทุกครั้ง
