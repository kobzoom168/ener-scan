# กติกาโปรเจกต์ (โหลดอัตโนมัติทุก session)

## เริ่มงาน
อ่านตามลำดับก่อนแตะโค้ด: `AI-README.md` → `docs/ai/STATE.md` → `docs/ai/LOG.md` (3-5 รายการล่าสุด) → `docs/ai/BACKLOG.md`

## จบงาน (บังคับ — ห้ามข้าม)
ทุกครั้งที่งานเสร็จหรือหยุดกลางคัน:
1. เติมบันทึกท้าย `docs/ai/LOG.md` — วันที่ | ใครทำ | ทำอะไร | ผล | ค้างอะไร
2. สถานะระบบเปลี่ยน (feature LIVE / env ใหม่ / กติกาใหม่) → อัปเดต `docs/ai/STATE.md`
3. งานในคิวขยับ → อัปเดต `docs/ai/BACKLOG.md`
4. commit ไฟล์บันทึกไปพร้อมงาน

## ความปลอดภัย
- ห้ามอ่าน/พิมพ์ค่าจริง `.env` prod (ดูได้แค่ชื่อตัวแปร)
- deploy pro เฉพาะเมื่อกบสั่งชัด ๆ · ลบ/แก้ข้อมูลลูกค้าต้องให้กบยืนยัน
- ห้าม `git checkout -- .` / ห้าม commit compose บนเครื่อง server
