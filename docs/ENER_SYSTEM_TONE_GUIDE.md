# Ener System Tone Guide

**Purpose:** ทุกข้อความในระบบ (สรุป ลักษณะพลัง คำอ่าน การใช้ ปิดท้าย) ต้องรู้สึกว่ามาจาก **เสียงเดียวกัน** — ชัด อบอุ่น ติดดิน ไม่สับสน

**Scope:** ใช้เป็นเกณฑ์หลักสำหรับ copy ใน `src/services/flex/scanCopy.config.js` (assembler: `scanCopy.generator.js`) และคำแนะนำสำหรับข้อความจากโมเดล (`deepScan.prompt.js`)  
**ไม่เปลี่ยน:** layout, routing, scoring

---

## Core tone

| ต้องเป็น | หลีกเลี่ยง |
|----------|------------|
| ภาษาไทยล้วน | ภาษาอื่นปน (ยกเว้นชื่อเฉพาะที่จำเป็น) |
| ชัด เข้าใจทันที | คำศัพท์เชิงพิธีกรรม/คลุมเครือจนอ่านไม่ตรง |
| อบอุ่น ติดดิน | หวือเกินไปจนสับสน |
| พูดแบบคนจริง | รายงาน / โฆษณา / หุ่นยนต์ |

---

## Global rules

1. **เข้าใจความหมายทันที** — อ่านครั้งเดียวรู้ว่าหมายถึงอะไรในชีวิตจริง  
2. **ใช้ภาษาชีวิตจริง** — ไม่ใช้ป้ายศัพท์เชิงพลังแบบลอย ๆ โดยไม่มีบริบท  
3. **กล่องสั้น / ป้ายกระชับ** = ภาษาไทยสั้น ๆ ตรง ๆ  
4. **กล่องกลาง / ประโยคสั้น** = ประโยคธรรมชาติ อ่านลื่น  
5. **ส่วนอ่านยาว** = อบอุ่น ติดดิน ลงลึกได้เล็กน้อย แต่ยังไม่หวือ  
6. **ห้ามใช้คำ/วลีที่ทำให้สับสนหรือฟังเก่า** เช่น:
   - `พลังนิ่ง` (เป็นป้าย)
   - `โทนทอง` (เป็นวลีเด่น)
   - `ผู้พิทักษ์` (บทบาทแฟนตาซี)
   - `ให้ความรู้สึกเหมือน...` (เปรียบเทียบกำกวม)
7. **ชอบใช้โครงประโยคนี้ใน product copy:** `ช่วยให้…` · `เหมาะกับ…` · `ทำให้…`

## Energy personalities (ความหมายที่ต้องสื่อ — ไม่ใช่คำบังคับทุกบรรทัด)

| พลังหลัก | ทิศทางเสียง |
|----------|-------------|
| **ปกป้อง** | นิ่งทนแรงกดดัน · หนุนหลังให้มั่น |
| **สมดุล** | คุมจังหวะ · ลดวุ่นในใจ |
| **อำนาจ** | มั่นใจ · ตัดสินใจ · ยืนหยัดในที่ประชุม |
| **เมตตา** | อ่อนโยน · เข้าถึงคน · บทสนทนานุ่มลง |
| **ดึงดูด** | น่าเข้าหา · โดดเด่นแบบไม่ฝืน |
| **โชคลาภ** | โอกาส · จังหวะ · การเลือกอย่างรอบคอบ |
| **เสริมพลัง** | กำลังใจ · ลุยต่อ · ไม่หมดแรงเร็ว |

## Where this applies in code

| พื้นที่ | แหล่งหลัก |
|---------|-----------|
| สรุป (หัวข้อใต้คะแนน, การ์ดพลังหลัก 2 บรรทัด) | `scanCopy.config.js` (`MAIN_LABEL`, `ENERGY_COPY` → `summary`) |
| ลักษณะพลัง 3 บรรทัด (feel / use / effect) | `scanCopy.config.js` (`ENERGY_COPY` → `traits`) + `scanCopy.utils.js` (pick / cap) |
| คำอ่าน (ภาพรวม / เหตุผล / ปิดท้าย) | จากโมเดล + `flex.display.js` — โมเดลต้องทำตามคู่มือนี้ |
| การใช้ (หัวข้อย่อย + bullets) | จากโมเดล + default ใน `flex.components.js` |
| Fallback เมื่อไม่มี `scanCopy` | `getEnergyShortLabel` ใน `flex.utils.js` |
